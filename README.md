# Homework 2 - Customer Support Response Drafting

## Business Workflow
This project focuses on drafting customer support responses for an e-commerce business.

## User
The user is a customer support agent who needs to respond to customer emails quickly and consistently.

## Input
The system receives a customer message, such as a complaint, refund request, delayed delivery inquiry, or damaged-item report.

## Output
The system produces a professional first-draft response email that addresses the customer’s issue, acknowledges the concern, and suggests a next step.

## Why This Task Is Valuable
This task is valuable to automate because support teams spend a lot of time writing repetitive first-pass responses. A draft generated by an LLM can save time, improve consistency, and allow human agents to focus on more complex cases.

## Prototype Implementation
This prototype uses a command-line Python script to run a stable evaluation workflow and save structured outputs. Customer-support replies are generated with Codex after signing in with ChatGPT, then recorded by the script in `outputs.json`.

## Reproducibility
A grader can reproduce the workflow by running `python3 app.py`, using the prompts documented in `prompts.md`, and entering generated responses for the cases in `eval_set.json`.
